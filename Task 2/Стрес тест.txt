## Виявлена мінімальна комбінація параметрів, що викликає відмову

При проведенні стрес-тестування було виявлено, що вебсервіс JSONPlaceholder починає демонструвати ознаки деградації продуктивності та відмов при наступних параметрах навантаження:

* Кількість віртуальних користувачів: 240
* Ramp-up period (секунд): 40
* Loop Count (кількість ітерацій на користувача): 2

Основні ознаки деградації/відмови, що спостерігались:

1. Зростання часу відповіді та помилки Duration Assertion для POST Request:1254 м/сек
    Для запиту PUT Request, за цих умов було виконано 276 семпли, з яких 0,36% завершились помилкою.
    Аналіз View Results in Tree показав, що ці помилки були спричинені спрацюванням Duration Assertion. Середній час відповіді склав 175 мс, але максимальний час 1254 мсм, що значно перевищує встановлений поріг у 1000 мс. Це свідчить про те, що API перестає швидко обробляти PUT запити під таким навантаженням.


 2.  Зростання часу відповіді та помилки Duration Assertion для PUT Request:1297 м/сек
